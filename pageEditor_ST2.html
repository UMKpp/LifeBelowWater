Home Page

Technical Description

The Home Page was developed using modern HTML5 and CSS3 techniques to create an engaging and immersive user experience that showcases the Ocean Conservation Initiative. Key HTML elements include:

  • Header with video background: Using the HTML5 <video> element with autoplay, loop, and muted attributes to create a dynamic ocean backdrop.
  • Interactive logo: Created with CSS animations using @keyframes to implement a 3D rotation effect (axis-spin) on hover.
  • Content cards: The design applies CSS Grid (grid-template-columns) for building a flexible two-column structure.
  • SDG Targets carousel: Implemented with JavaScript to aid in control of horizontal scrolling through target cards.
  • Publications section: Used relative and absolute positioning to create overlay effects for "Read more" buttons and on hover attribute is used to make the button visible.

CSS techniques include:

  • CSS transitions and transforms for smooth hover effects on cards and buttons
  • Linear gradients for background color transitions
  • CSS Grid and Flexbox for responsive layouts
  • Media queries to ensure mobile responsiveness
  • Hover attributes for interactive display
  • AOS (Animate On Scroll) library integration for scroll-triggered animations

Accessibility

Accessibility features implemented on the Home Page include:

  • Semantic HTML5 elements (<header>, <section>, <nav>, <footer>) to support screen readers
  • Alt text for all images, including the logo with an appropriate aria-label
  • Color contrast that meets WCAG guidelines between text and background
  • Keyboard navigable elements for all interactive components
  • The video component includes alternative text that displays for browsers which do not support video playback.
  • Focus indicators for interactive elements

Table Page

Technical Description

The Table Page was designed to present organized data about some of the endangered species of marine animals. Key HTML and CSS elements include:

  • Responsive tables: Created with CSS media queries to enforce readability on various screen sizes
  • Zebra striping: Used CSS nth-child selectors to create alternating row colors for better readability
  • Fixed headers: Implemented sticky positioning for table headers to maintain visibility during scrolling
  • Brand Enforcement The logo of the website is highlighted prominently for brand enforcement

Notable CSS properties used:

  • table-layout: fixed - For consistent column widths
  • border-collapse: separate - For clean table appearance
  • overflow-x: auto - To enable horizontal scrolling on small screens
  • position: sticky - For fixed headers during scrolling
  • box-shadow and border-radius - For elevated design elements
  • on-hover attribute is used to show interactivity with user
  • tooltip features for image description

Accessibility

Accessibility considerations for the Table Page include:

  • Proper table markup with <thead>, <tbody>, and <th scope="col/row"> for screen readers
  • ARIA attributes for sortable columns to indicate their interactive nature
  • Caption elements to provide context for each table
  • Sufficient color contrast between cell content and backgrounds
  • Keyboard navigation support for table interaction
  • Responsive design to avoid horizontal scrolling when possible

Content Page - Marine Mammals

Technical Description

The Marine Mammals content page was designed to provide educational information about diverse marine mammal species. Key HTML and CSS techniques include:

  • content section: Implemented with a large, responsive image and overlay text
  • Card-based content layout: Used flexbox for responsive organization of information cards
  • Image galleries: Created with CSS Grid for organized display of species images
  • Pull quotes: Used CSS positioning and styling to highlight important facts

Notable CSS properties used:

  • CSS Grid and Flexbox for responsive layouts
  • CSS transitions for smooth animations when revealing content
  • Object-fit: cover - To maintain image aspect ratios
  • AOS (Animate On Scroll) library integration for scroll-triggered animations
  • Background blend modes for creative image treatments

Accessibility

Accessibility features implemented on the Content Page include:

  • Proper heading hierarchy (h1-h6) for logical document structure
  • Alt text for all images describing marine mammals
  • ARIA attributes for interactive elements
  • Keyboard accessible navigation for all interactive components
  • Focus indicators for interactive elements
  • Sufficient text spacing for readability
  • Responsive design that works well with different screen sizes

Challenges and Lessons Learned

Throughout the development process of these pages, I encountered several significant challenges:

Challenge 1: Responsive Carousel Implementation

The SDG Targets carousel on the homepage presented difficulties with responsive behavior. Initial implementation resulted in layout issues on mobile devices, with cards overlapping and buttons becoming difficult to tap.

Solution: I refactored the JavaScript to dynamically calculate card widths based on viewport size and implemented touch events for mobile users. This taught me the importance of testing across different devices early in development.

Challenge 2: Table Responsiveness

Making data tables responsive while maintaining usability was challenging. Initially, tables would overflow on small screens, creating horizontal scrolling issues.

Solution: I implemented a combination of techniques: collapsible rows for less important data, horizontal scrolling containers for tables with many columns, and responsive design patterns that display data differently on small screens. This taught me to prioritize content and think about how data can be restructured for different contexts.

Challenge 3: Maintaining Text Readability on Overlay Images

Some sections of the content page, such as the introduction and key species, used background images with overlay text. Initially, the text was difficult to read due to poor contrast against the images.

Solution: I applied a semi-transparent dark overlay using `background-blend-mode` and `rgba` backgrounds. Additionally, I adjusted text shadow and font weight to enhance readability without sacrificing design aesthetics. This taught me the importance of contrast and accessibility when layering text over images.

Challenge 4: Global file styles conflicting with the individual styles of the page

The styles like the margin and padding etc, started interfering with the page styles causing the format of the pages to go awry.

Solution: I changed specified the classes and the elements more specifically so that their styles don't intefere with eachother. This taught me the need to define classes in a very specific manner, so that styles don't intefere with eachother

Key Lessons Learned:

  • Cross-browser testing is essential - solutions that work in one browser might fail in others
  • Progressive enhancement is better than assuming all users have high-end devices
  • Accessibility cannot be an afterthought - it must be built in from the beginning
  • Performance optimization is crucial for mobile users
  • Classes used for styling must be well defined

Compliance

Content Compliance

  • All content is educational in nature and relates directly to the SDG 14: Life Below Water theme
