Splash Page

Technical Description

Successfully designed and implemented a splash screen for our website. It displays a statement and group members' names to promote transparency and community engagement. And implemented automatic redirection to the Home page after 4 seconds and an animated loader image to enhance user experience and navigation.

Accessibility

When implementing the splash page, I took several measures to ensure accessibility, making the page more inclusive and usable for all users.

  1. Used appropriate HTML tags such as h1, h3, and p to provide a meaningful document structure, making it easier for screen readers to navigate the content.
  2. Used (video) for background media, ensuring it is properly structured within a (div) for clarity.
  3. Included a fallback message (Your browser does not support the video tag.) in case the video does not load or the browser does not support it.
  4. The loader animation does not rely on flashing or high-speed transitions that could cause discomfort to users with motion sensitivity.
  5. The structured HTML allows screen readers to read the page in a logical order.

Volunteer Page

Technical Description

This volunteer opportunities webpage by SeaScape provides a platform for individuals passionate about marine conservation. It features an interactive sorting feature, and detailed volunteer program cards with images, ratings, and descriptions. Users can explore various programs, read reviews, and submit feedback. A well-structured footer includes company details, social media links, and additional navigation options

Accessibility

Explain how you used to addressed accessibility in this particular page you implemented.

  1. Proper use of header, nav, section, and footer elements helps assistive technologies, like screen readers, to navigate the content efficiently. Additionally, appropriate heading levels (h2, h3) provide a clear document hierarchy.
  2. The feedback form includes a clear placeholder and labels, making it easier for screen readers to interpret form fields.
  3. All images, including logo and volunteer program images, have alt attributes to provide meaningful descriptions for visually impaired users.
  4. The page uses high contrast text and background colors, making content easier to read for users with visual impairments.
  5. ARIA attributes (if needed in future enhancements) can further improve accessibility for dynamic content, such as badges and icons used in the volunteer cards.

Content Page

Technical Description

The Rare Fish Collection webpage is a well-structured and responsive HTML document that presents information about rare fish species. It includes navigation elements, content sections, and a footer, ensuring a user-friendly experience. The webpage is designed using HTML, CSS, and external resources to maintain an aesthetically appealing and accessible layout.

Accessibility

Explain how you used to addressed accessibility in this particular page you implemented.

  1. The page structure follows a semantic HTML approach, using elements like (header), (nav), (main), (section), (footer), and (article), which improves screen reader compatibility and helps assistive technologies understand the content hierarchy.
  2. Every image (img) in the page has an alt attribute, describing the image content. This ensures that. Screen readers can describe images to visually impaired users.If images fail to load, users can still understand what they represent.
  3. All interactive elements such as navigation links, buttons, and "Read More" links can be accessed using only the keyboard (Tab key navigation).
  4. The color scheme ensures sufficient contrast between the text and background to help users with visual impairments read content easily.
  5. The contact information is presented in plain text, ensuring it is easily readable and accessible by screen readers.

Challenges and Lessons Learned

Throughout the development of the splash screen, volunteer page, and content page, I encountered challenges such as ensuring cross-browser compatibility, optimizing performance while maintaining aesthetics, and making interactive elements accessible to all users. Implementing semantic HTML, keyboard navigation, high contrast designs, and alternative content (alt text, fallbacks) significantly improved accessibility. I learned that accessibility should be an integral part of the design process from the start, not an afterthought. Testing across different devices, browsers, and assistive technologies is essential to create an inclusive and user-friendly experience.

Compliance

  1. The pages follow WCAG 2.1 guidelines, ensuring proper semantic HTML, alt attributes for images, and keyboard navigability. Structured navigation, readable fonts, and high-contrast colors improve inclusivity for users with disabilities.
  2. No sensitive user data is collected or stored, aligning with GDPR (General Data Protection Regulation) principles. External links are properly referenced, minimizing security risks. The website does not use unauthorized tracking scripts or intrusive pop-ups.
  3. The information presented is factual, sourced from credible references, and does not include misleading or offensive content. Citations for external sources are provided where necessary.
  4. The website is structured using validated HTML and CSS, ensuring compliance with web standards. Responsive design principles are applied, allowing for smooth adaptation across various devices and screen sizes.
