Challenges and Lessons Learned

One of the main challenges was ensuring accessibility compliance, including keyboard navigation, screen reader support, and high contrast design. Implementing form validation for the feedback page was also crucial to prevent incorrect inputs. Additionally, achieving cross-browser compatibility required testing and layout adjustments. Through this process, I learned the importance of prioritizing accessibility from the start, thorough testing across browsers, and enforcing proper validation for usability and security. These challenges helped me enhance my problem-solving skills and understanding of web standards.

Compliance

Compliance The Feedback Page, Team Page, and Regional Seas Content Page comply with web standards, accessibility guidelines, and security best practices. Web Standards: Pages were validated using W3C tools, ensuring proper semantic HTML and responsive CSS. Accessibility: Implemented keyboard navigation, screen reader support, high contrast design, and alt text for images to meet WCAG 2.1 guidelines. Security & Privacy: Form validation prevents malicious inputs, and no sensitive user data is stored. Cross-Browser Compatibility: Tested across major browsers (Chrome, Firefox, Edge) for consistency.
